04 基于qemu搭建op-tee运行环境

2025-02-28

TrustZone TEE ARMv8 QEMU op-tee

#111 介绍了如何在QEMU上模拟SoC启动ATF，以及分析了从BootROM到ATF，最后启动u-boot的过程。TEE的启动在 #111 中并没有实现，因为TEE OS的启动需要涉及ATF对SoC的内存划分，以及启动时序等。不过，OP-TEE有基于QEMU的ARMv8的全套模拟方案，可以实现BL1、BL2、BL31、BL32、BL33和Linux runtime，OP-TEE QEMU v8.

克隆代码

  • 工具安装

    • OP-TEE使用的是git-repo管理仓库，在下载代码之前，需要先安装repo工具：

    ``bash

    $ git clone -b v2.12.2 http://mirrors.ustc.edu.cn/aosp/git-repo.git

    $ chmod u+x git-repo/repo

    $ mkdir -p ~/bin

    $ cp git-repo/repo ~/bin

    $ echo "export PATH=~/bin:$PATH" >> ~/.zshrc # or ~/.bashrc

    $ source ~/.zshrc

    `

  • 下载OP-TEE QEMU仿真环境代码
    • `bash

    $ mkdir optee

    $ cd optee

    $ repo init -u https://github.com/OP-TEE/manifest.git -m qemu_v8.xml

    $ repo sync -j8

    `

    编译和运行

    OP-TEE仿真平台提供了一键式编译仓库build，它可以自动下载对应的交叉编译器，我们的编译和运行也都是在这个仓库里完成：

  • 编译
    • `bash

    make -j8 toolchains # 下载和编译交叉编译器

    make -j8 # 编译

    ···

  • 运行

    • 运行就是基于QEMU，运行前面编译好的固件，因此我们使用qemu_v8的配置文件来运行：

    `bash

    make -f qemu_v8.mk run-only

    ``

    执行运行后，就可以看到三个terminal窗口：

    * make -f qemu_v8.mk run-only：原始命令行terminal，启动了一个gdb调试

    * Normal World：REE侧，运行这buildroot编译出来的runtime

    * Secure World： TEE侧安全世界

    在gdb侧输入c，按回车键，可以触发Normal World和Secure World启动：
