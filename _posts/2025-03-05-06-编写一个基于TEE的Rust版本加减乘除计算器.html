为什么要使用Rust编写TA/CA

我们通过前面的例子已经看到，OP-TEE使用C语言已经可以无故障开发TA/CA了，并且C语言相比其他语言，效率和性能也更高，那么为什么还要引入Rust到OP-TEE的TA/CA开发呢？我们可以看两个例子：

  • Qualcomm's Secure Execution Environment(QSEE) privilege escalation vulnerability and exploit: CVE-2015-6639
  • Extracting Qualcomm's KeyMaster Keys - Braking Android Full Disk Encryption:

    • 以上两个例子，都是利用了C语言的内存安全问题，打破了TrustZone的安全保证，泄漏用户的秘钥或者是敏感数据。为了解决TrustZone的安全机制被内存漏洞打破的问题，Rust从2018年开始，逐步引入到OP-TEE.

    Rust如何引入到OP-TEE

    在OP-TEE的架构中，EL1层运行了REE OS的驱动和TEE OS，他们分别运行在Normal World和Secure World。基于REE OS的OP-TEE驱动，OP-TEE开发了TEE Client SDK，这个SDK遵循GlobalPlatform TEE的API，其他自定义的TEE厂商也会遵循这个API规范，开发满足GlobalPlatform TEE规范的SDK。基于Secure World的OP-TEE Trusted OS，OP-TEE开发了TEE TA SDK，然后基于这个SDK，用户可以自行编写自己的TA，TEE TA SDK同样是满足GlobalPlatform OP-TEE Internal Core C API规范的。由于这两个SDK都是使用C语言编写，一旦内存指针管理不善，就可能导致Secure World的敏感信息泄露！

    为了在基于TrustZone平台的芯片上支持Rust编写TA和CA，需要使用Rust编写Rust OP-TEE TrustZone SDK，用于替换原始的TEE TA SDK和TEE Client SDK：

    在Secure World，需要基于Rust编写一些底层的适配库，这些库仍然是基于C语言编写的libutee和libutil实现，然后用Rust调用C，并封装实现，提供一个optee-utee，提供给上层TA调用：

    在Normal World，也是基于libteec进行Rust封装，提供接口生成optee-teec，提供给上层CA调用：

    编写一个例子

    在前面 #113 ，我们使用C语言编写了一个基于TEE的加减乘除计算器，实际上，OP-TEE目前已经支持Rust了，有一个专门的目录optee-rust专门用来存放使用Rust编写的代码和例子。我们现在基于optee-rust的架构，同样编写一个加减乘除计算器！

    首先观察Rust的TA/CA的架构：

    修改UUID

    Rust的TA/CA的UUID是通过uuid.txt传递的，proto库中通过一个常量将UUID保存下来：

    `Rust

    // proto/src/lib.rs

    pub const UUID: &str = &include_str!(concat!(env!("OUT_DIR"), "/uuid.txt"));

    `

    这样，TA和CA引用proto库就可以获取这个UUID的值

    修改proto

    proto里定义了TA和CA会话的Command，我们在这里定义加减乘除：

    `Rust

    pub enum Command {

    IncValue,

    DecValue,

    Multiple,

    Divide,

    Unknown,

    }

    impl From for Command {

    #[inline]

    fn from(value: u32) -> Command {

    match value {

    0 => Command::IncValue,

    1 => Command::DecValue,

    2 => Command::Multiple,

    3 => Command::Divide,

    _ => Command::Unknown,

    }

    }

    }

    `

    这里只是例子，因此我们只实现了u32类型的数据的加减乘除。

    use optee_teec::{Context, Operation, ParamType, Session, Uuid};

    use optee_teec::{ParamNone, ParamValue};

    use proto::{UUID, Command};

    fn operate(session: &mut Session, command: Command, lhs: u32, rhs: u32) -> optee_teec::Result<()> {

    let p0 = ParamValue::new(lhs, 0, ParamType::ValueInout);

    let p1 = ParamValue::new(rhs, 0, ParamType::ValueInput);

    let mut operation = Operation::new(0, p0, p1, ParamNone, ParamNone);

    println!("Operating {:?} and {:?} ...", operation.parameters().0.a(), operation.parameters().1.a());

    session.invoke_command(command as u32, &mut operation)?;

    println!("Result is {:?}", operation.parameters().0.a());

    Ok(())

    }

    fn main() -> optee_teec::Result<()> {

    let args: Vec = std::env::args().collect();

    if args.len() != 4 {

    eprintln!("Usage: {} ", args[0]);

    std::process::exit(1);

    }

    let lhs: u32 = args[1].parse().expect("First num is not valid!");

    let op = &args[2];

    let rhs: u32 = args[3].parse().expect("Second num is not valid!");

    let operator = match op.as_str() {

    "+" => Command::IncValue,

    "-" => Command::DecValue,

    "*" => Command::Multiple,

    "/" => Command::Divide,

    _ => Command::Unknown,

    };

    let mut ctx = Context::new()?;

    let uuid = Uuid::parse_str(UUID).unwrap();

    let mut session = ctx.open_session(uuid)?;

    operate(&mut session, operator, lhs, rhs)?;

    println!("Success");

    Ok(())

    }

    #![no_std]

    #![no_main]

    #![feature(c_size_t)]

    use optee_utee::{

    ta_close_session, ta_create, ta_destroy, ta_invoke_command, ta_open_session, trace_println,

    };

    use optee_utee::{Error, ErrorKind, Parameters, Result};

    use proto::Command;

    #[ta_create]

    fn create() -> Result<()> {

    trace_println!("[+] TA create");

    Ok(())

    }

    #[ta_open_session]

    fn open_session(_params: &mut Parameters) -> Result<()> {

    trace_println!("[+] TA open session");

    Ok(())

    }

    #[ta_close_session]

    fn close_session() {

    trace_println!("[+] TA close session");

    }

    #[ta_destroy]

    fn destroy() {

    trace_println!("[+] TA destroy");

    }

    #[ta_invoke_command]

    fn invoke_command(cmd_id: u32, params: &mut Parameters) -> Result<()> {

    trace_println!("[+] TA invoke command");

    let mut values = unsafe { params.0.as_value().unwrap() };

    let rhs = unsafe { params.1.as_value().unwrap() };

    match Command::from(cmd_id) {

    Command::IncValue => {

    values.set_a(values.a() + rhs.a());

    Ok(())

    }

    Command::DecValue => {

    values.set_a(values.a() - rhs.a());

    Ok(())

    }

    Command::Multiple => {

    values.set_a(values.a() * rhs.a());

    Ok(())

    }

    Command::Divide => {

    values.set_a(values.a() / rhs.a());

    Ok(())

    }

    _ => Err(Error::new(ErrorKind::BadParameters)),

    }

    }

    // TA configurations

    const TA_FLAGS: u32 = 0;

    const TA_DATA_SIZE: u32 = 32 * 1024;

    const TA_STACK_SIZE: u32 = 2 * 1024;

    const TA_VERSION: &[u8] = b"0.1\0";

    const TA_DESCRIPTION: &[u8] = b"This is a hello world example.\0";

    const EXT_PROP_VALUE_1: &[u8] = b"Hello World TA\0";

    const EXT_PROP_VALUE_2: u32 = 0x0010;

    const TRACE_LEVEL: i32 = 4;

    const TRACE_EXT_PREFIX: &[u8] = b"TA\0";

    include!(concat!(env!("OUT_DIR"), "/user_ta_header.rs"));
